In this lesson we will talk about transaction input validation.

Transaction input validation is extremely important. One common mistake that can be seen is that the transaction amount checks are similar to: 

if tx.quantity <= wallet.balance:
        # transaction ok
else:
        # reject transaction
However, with such a check, one can create a transaction from oneself to oneself with a negative amount, which won't be caught by the check above. Thus creating coins out of thin air: 
    Transaction id: 1234
    Source: myself
    Destination myself
    Amount: -9999999
A simple protection against this is to make sure that transaction amounts are greater than zero.

To add a new FumbleStore lesson, all changes need to be performed on the fumblestore git branch of the fumblechain repository. 

$ cd fumblechain
$ git checkout fumblestore
$ cd src/fumblechain

First, create a new HTML file in the templates/lessons directory. This file's contents will be displayed when the new lesson is selected. For example, assume we create a new file templates/lessons/my-new-lesson.html

Next, add the reference to your new lesson HTML file in store/lesson.py in the get_available_lessons() function. This function returns a dictionary of lesson category names to list of lessons. Each lesson is an object of type Lesson which takes 3 arguments:

  • The lesson ID, which must be a unique integer for all lessons.
  • The name of the lesson, a string.
  • The path to the lesson's HTML file, relative to the templates directory.

Full example: 

def get_available_lessons():
    lessons = {
        "Category A": [
            Lesson(1, "Introduction to FumbleChain", "lessons/fumblechain.html"),
            Lesson(2, "Using the FumbleChain CLI", "lessons/fumblechain-cli.html"),
            Lesson(3, "Using the Blockchain explorer", "lessons/blockchain-explorer.html")
        ],
        "Category B": [
            Lesson(1000, "What is a blockchain?", "lessons/what-is-a-blockchain.html"),
            Lesson(1001, "MY NEW LESSON", "lessons/my-new-lesson.html")
        ]
    }

    return lessons
